# Context Index

Orden de lectura obligatorio para cualquier persona o agente que vaya a iterar en Alvear:

1. `docs/context/project_brief.md`
2. `docs/context/current_vs_target_architecture.md`
3. `docs/context/implementation_snapshot.md`
4. `docs/context/offline_adaptation_spec.md`
5. `docs/context/test_scenario_v1.md`
6. `docs/context/decisions.md`
7. `docs/context/worklog.md`
8. `docs/context/upstream_provenance.md`

## Que responde cada documento

- `project_brief.md`: que es Alvear, para quien existe, como debe sonar y que buscamos validar primero.
- `current_vs_target_architecture.md`: donde estamos respecto a MiroFish y como se traduce eso a un backend offline.
- `implementation_snapshot.md`: que codigo existe hoy, que comandos funcionan ya y que piezas siguen bloqueadas por dependencias o servicios locales.
- `offline_adaptation_spec.md`: contratos tecnicos canonicos de v1, incluida la reconciliacion de estado y la capa de reporting humano.
- `test_scenario_v1.md`: seed y criterios de aceptacion para la primera prueba oficial.
- `decisions.md`: decisiones ya tomadas, con fecha.
- `worklog.md`: estado real del repo, siguiente paso y bloqueos.
- `upstream_provenance.md`: trazabilidad de reutilizacion desde MiroFish.
