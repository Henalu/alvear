# Context Index

Orden de lectura obligatorio para cualquier persona o agente que vaya a iterar en Alvear:

1. `docs/context/project_brief.md`
2. `docs/context/current_vs_target_architecture.md`
3. `docs/context/implementation_snapshot.md`
4. `docs/context/offline_adaptation_spec.md`
5. `docs/context/test_scenario_v1.md`
6. `docs/context/decisions.md`
7. `docs/context/worklog.md`
8. `docs/context/upstream_provenance.md`

## Qué responde cada documento

- `project_brief.md`: qué es Alvear, para quién existe, cómo debe sonar y qué buscamos validar primero.
- `current_vs_target_architecture.md`: dónde estamos respecto a MiroFish y cómo se traduce eso a un backend offline.
- `implementation_snapshot.md`: qué código existe hoy, qué comandos funcionan ya y qué piezas siguen bloqueadas por dependencias o servicios externos locales.
- `offline_adaptation_spec.md`: contratos técnicos canónicos de v1.
- `test_scenario_v1.md`: seed y criterios de aceptación para la primera prueba oficial.
- `decisions.md`: decisiones ya tomadas, con fecha.
- `worklog.md`: estado real del repo, siguiente paso y bloqueos.
- `upstream_provenance.md`: trazabilidad de reutilización desde MiroFish.
